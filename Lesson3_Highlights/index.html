Lesson 3 Highlights

Daniel Vincent

Highlight #1 Basics of Version Control

Version control is something that I have never really known a lot about, so it was really interesting to learn more about it in this lesson. As I was going through the readings and learning about version control, I kept thinking to myself, “Oh, that’s such a good idea!” It had just never occurred to me that there could be such a good system for keeping track of changes made across different versions of a project. It’s clear to me now that version control is an incredibly important part of creating in the 21st century, and it’s about time that I learned more about it.

Highlight #2 Version Control in Writing Projects

My primary major is English, and learning about version control immediately made me think about the importance of version control in writing projects. Throughout my major, I’ve had a variety of collaborative writing projects, with anything from short papers to creating entire websites together. Our default platform for creating collaborative writing projects is Google Docs, because everyone can access the project with Internet access, but this lesson made me see that a full, more robust VCS system like Github would be even more helpful. Github would allow my writing partners and I to even better track changes that were made, and would provide a much more secure system to track our writing over time.

Highlight #3 Version Control in Musical Projects

In addition to writing projects, this lesson on VCS made me think of the importance of version control in musical production projects. My brother and I are both musicians and we’ve done a few recordings of songs we’ve written. Though we’ve done most recordings together in the same place, there have been a couple where we’ve had to trade recordings back and forth and work on the same files. With a proper use of version control, we could make that process so much quicker and much easier to track changes. By using a good version control system, it would be so much easier to work on our recordings back and forth, because we could upload the projects and wouldn’t have to worry about messing with each others’ edits.

Highlight #4 Remote vs. Local Repositories

This lesson also showed me the importance of keeping remote repositories of information instead of just keeping local repositories of information. From my own understanding of remote repositories before this lesson, I knew they were important, but my only understanding of remote storage was to make sure there was a backup saved somewhere in case my local storage went down. But, this lesson showed me that remote repositories are also an important part of any project because they allow others to have equal access to the information that I would upload to the remote repository. The way in which version control systems allow this sort of sharing between users on the network allows for better collaboration and better communication throughout the creation of a project.

Highlight #5 Git and Github

One of the most interesting parts of this lesson to me was the use of Git to do version control and Github as our remote repository. It seems like a widely used and widely accepted version control system to use, and it fits all of our needs in this class perfectly. But, it wasn’t until I watched the second video from Github that I realized that Git was not the only version control system out there. It just made me think: are there any other version control systems that might work better for certain projects? For example, could there be a different version control system that would work well for a musical project or a video project? While I think Git is probably the best thing that I could find for version control, watching that video just made me think about any other possibilities for version control systems.
